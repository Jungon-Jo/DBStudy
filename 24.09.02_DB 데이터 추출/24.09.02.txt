	1. 점수가 50미만인 회원수를 추출하는 말을 만드시오.
	→ hm relation에서 column의 point 값이 50 미만의 값을 가지는 튜플을 출력한다.
	→ hm relation을 참조하여 50미만인 조건을 column에 걸고 집계함수를 사용하여 튜플의 갯수를 출력한다.
	
	2. 점수가 50미만인 회원수 중에 점수가 높은 순서대로 회원수를 출력
	→ hm relation을 참조하여 50미만인 조건을 column에 걸고 point 값이 높은 순서대로 정렬하여 튜플을 출력한다.

	함수 : 기능을 가졌다. 함수 호출 → 리턴값이 발생한다.

	1. 테이블 관리
	: create table, drop table, alter table
	: 테이블 relation, 속성(타입, 제약조건) attribute
	: 타입 - int, number*, varchar2, timestamp
	: 제약조건 - 유일성(기본키, unique), not null, check, default
	: 키	기본키	: 유일성, not null, 인덱스 형성(검색 속도 향상 = where 절에서 사용)
		외래키	: 컬럼 투 컬럼, 자식 relation column이 부모
					: 외래키 제약 조건, cascade, set null
	: ERD	개체와 개체의 관계 분석 1:1, 1:N, N:M 분석 후 테이블 형성

	2. 테이블에 저장된 튜플의 조작
	: 삽입 - insert
	: 갱신 - update → 특정 튜플을 선택하여 갱신하려면 where 절 사용


	: 삭제 - delete → 특정 튜플을 선택하여 삭제하려면 where 절 사용
	: 검색 - select → 각 절의 의미 암기, 절의 실행 순서 이해
	: 실행 순서/절의 의미
		4. select (컬럼 선택)
		1. from (대상 테이블)
		2. where (대상 테이블로 부터 튜플을 선정할 조건)
		3. group by (그룹화)
		5. order by (출력할 때 정렬)

	3. 함수 :	함수를 호출하고 함수의 기능을 실행하고 결과값을 리턴,
		결과값을 리턴받아서 처리한다.
		예를 들어 count(*)함수를 호출하면 튜플의 수를 카운팅하여 카운팅 결과값을 리턴한다.
